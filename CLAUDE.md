## Claude Code 专属说明

- **最高优先级**：`AGENTS.md` 中的 §4 关键约束 与 §12 务必/禁止 不可违背。如发现某项需求与之冲突，先停下并向用户确认，不要擅自变通。
- **逐里程碑推进**：默认按 `AGENTS.md` §11 的 M1→M7 顺序工作。开始某里程碑前，先用一段话列出本里程碑要触碰的文件清单，经确认后再写代码。
- **接口契约优先**：涉及界面时，先确保 `render_ui` 工具 schema（§6.1）与 `packages/ui_spec` 的 JSON Schema 一致；二者任一变更必须同步另一方。
- **不确定即核实**：遇到带 `⚠️待核实` 的外部命令或版本号，先联网查官方文档再落地，禁止臆造。
- **变更后自检**：每次提交前运行对应模块的 lint 与测试（§9），并自查是否泄露了任何 secret。
- **小步提交**：遵循 Conventional Commits，一次提交聚焦一个可验证的改动。
